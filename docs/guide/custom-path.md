# 自定义游戏路径

MCLa 支持自定义 `.minecraft` 游戏目录的存储路径。本章将介绍如何设置自定义路径、启用版本隔离、实现多启动器共存以及迁移已有的游戏数据。

## 设置自定义路径

默认情况下，MCLa 将游戏文件存储在系统用户目录下的 `.minecraft` 文件夹中。你可以将其更改为任意位置，例如独立的磁盘或分区。

### 设置步骤

1. 进入 **设置** 页面
2. 找到「游戏目录」设置项
3. 点击 **「浏览」** 按钮
4. 在文件夹选择器中选择目标目录（可以选择已有目录或新建目录）
5. 点击「保存」
6. **重启 MCLa** 使设置生效

### 路径格式

| 路径类型 | 示例 | 说明 |
|---------|------|------|
| 绝对路径 | `D:\Games\.minecraft` | 推荐使用，明确指向目标位置 |
| 相对路径 | `./minecraft` | 相对于 MCLa 安装目录 |

::: warning 注意
- 路径中 **不应包含中文、空格或特殊字符**，否则可能导致游戏文件下载或启动失败。
- 建议将游戏目录放在 **SSD 固态硬盘** 上，以获得更快的加载速度。
- 修改游戏目录后，原有的游戏文件不会自动迁移，需要手动复制或通过实例导入功能迁移。
:::

## 版本隔离

版本隔离是管理多个游戏版本的重要机制。启用版本隔离后，每个游戏版本会在游戏目录下创建独立的子目录结构。

### 隔离目录结构

```
.minecraft/
  ├── versions/
  │   ├── 1.20.4/
  │   │   ├── mods/          # 该版本专属 Mod
  │   │   ├── config/        # 该版本专属配置
  │   │   ├── saves/         # 该版本专属存档
  │   │   └── resourcepacks/ # 该版本专属资源包
  │   └── 1.19.2/
  │       ├── mods/
  │       ├── config/
  │       ├── saves/
  │       └── resourcepacks/
  ├── assets/                # 共享资源文件
  └── libraries/             # 共享库文件
```

### 启用方式

- 使用 **实例系统** 创建的实例默认启用版本隔离
- 在「版本」页面也可以单独为某个版本启用隔离

## 多启动器共存

如果你同时使用多个 Minecraft 启动器（如 HMCL、PCL2），合理的路径规划可以避免文件冲突。

### 共存方案

| 方案 | 说明 | 推荐度 |
|------|------|--------|
| 独立游戏目录 | 为每个启动器设置不同的游戏目录 | 高度推荐 |
| 版本隔离 | 所有启动器共用一个游戏目录，但各自启用版本隔离 | 推荐 |
| 共用目录 | 多个启动器共用同一个游戏目录且不启用隔离 | 不推荐 |

### 与 HMCL / PCL2 共存建议

1. **为 MCLa 设置独立的游戏目录**（如 `D:\Games\MCLa\.minecraft`）
2. 其他启动器保持各自的游戏目录不变
3. 如果必须共用目录，确保所有启动器都启用版本隔离
4. 避免多个启动器同时运行，防止文件锁定冲突

::: warning 注意
- 不同启动器对版本隔离的实现方式可能不同，混用时可能出现意外行为。
- 建议不要在多个启动器中同时管理同一个游戏版本。
:::

## 迁移已有游戏数据

如果你之前使用其他启动器（如官方启动器、HMCL、PCL2）已经下载了游戏文件和存档，可以通过以下方式迁移到 MCLa。

### 方式一：直接指定游戏目录

1. 找到原有启动器的游戏目录位置（通常为 `~/.minecraft`）
2. 在 MCLa 的「设置」中将游戏目录指向该路径
3. 重启 MCLa
4. MCLa 会自动识别已有的游戏版本和存档

### 方式二：通过实例导入

1. 进入 **实例** 页面
2. 点击 **「导入」** 按钮
3. 选择原有的 `.minecraft` 目录
4. MCLa 会自动扫描并创建对应的实例
5. 导入完成后，原有的游戏文件保持不变

### 迁移注意事项

| 项目 | 说明 |
|------|------|
| 游戏版本 | MCLa 会自动识别已安装的游戏版本 |
| Mod 文件 | 如果启用了版本隔离，Mod 会随版本一起迁移 |
| 存档文件 | 存档会保留在原位置，MCLa 可以直接读取 |
| 启动器配置 | 其他启动器的配置不会迁移，需要在 MCLa 中重新设置 |
| ModLoader | 已安装的 ModLoader 会被自动识别 |

::: tip 提示
迁移前建议先 **备份** 原有的 `.minecraft` 目录，以防万一。复制整个目录到安全位置即可。
:::
