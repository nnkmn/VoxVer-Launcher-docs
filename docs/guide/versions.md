# 版本管理

VoxVer Launcher 提供完整的 Minecraft 版本管理功能，支持所有官方发布的版本类型，并集成四大主流 ModLoader 的一键安装。本章将详细介绍版本列表的使用、版本安装与删除、ModLoader 安装以及版本隔离机制。

## 版本列表

VoxVer Launcher 从 BMCLAPI 镜像源获取 Minecraft 官方版本列表，涵盖所有历史版本。

### 刷新版本列表

1. 进入 **版本** 页面
2. 点击 **「刷新版本列表」** 按钮
3. VoxVer Launcher 会从远程服务器拉取最新的版本元数据
4. 刷新完成后，版本列表会按类型分组显示

### 版本类型说明

| 版本类型 | 说明 | 稳定性 |
|---------|------|--------|
| 正式版（Release） | 经过完整测试的稳定版本 | 高，推荐使用 |
| 快照（Snapshot） | 下一个正式版的开发预览 | 中，可能存在 Bug |
| 远古版（Old Version） | Alpha / Beta 等早期历史版本 | 低，仅供怀旧体验 |
| 愚人节版（April Fools） | 每年 4 月 1 日发布的恶搞版本 | 低，仅供娱乐 |

## 安装版本

### 安装步骤

1. 在版本列表中找到你需要的版本
2. 点击该版本旁的 **「安装」** 按钮
3. VoxVer Launcher 开始下载游戏核心文件，包括：
   - 游戏主程序 JAR 文件
   - 游戏库文件（libraries）
   - 资源文件（assets）
   - 原生库文件（natives）
4. 下载进度会在右下角的全局下载弹窗中实时显示
5. 下载完成后，版本状态会更新为 **「已安装」**

::: info 说明
安装完成后，游戏文件会保存在游戏目录下的 `versions/<版本名>/` 文件夹中。首次安装通常需要下载 200MB - 500MB 的文件，具体取决于版本。
:::

## 版本隔离

版本隔离是 VoxVer Launcher 的重要特性，它允许每个游戏版本拥有独立的运行环境，避免不同版本之间的 Mod、配置和存档互相干扰。

### 隔离机制说明

| 隔离内容 | 说明 |
|---------|------|
| 游戏文件 | 每个版本有独立的 JAR 和 JSON 文件 |
| Mod 文件 | 每个版本有独立的 `mods/` 目录 |
| 配置文件 | 每个版本有独立的 `config/` 目录 |
| 存档数据 | 每个版本有独立的 `saves/` 目录 |
| 资源包 | 每个版本有独立的 `resourcepacks/` 目录 |

::: tip 提示
如果你使用 [实例系统](/guide/instances)，每个实例默认启用完整的版本隔离。你也可以在版本管理中单独为某个版本启用隔离。
:::

## 安装 ModLoader

ModLoader 是加载和管理游戏 Mod 的必要工具。VoxVer Launcher 支持一键安装四大主流 ModLoader。

### 安装步骤

1. 在「版本」页面找到已安装的原版游戏
2. 点击该版本旁的 **「安装 ModLoader」** 按钮
3. 在弹出的选择框中选择 ModLoader 类型
4. 选择对应的版本号（通常有「最新版」和「推荐版」可选）
5. 点击确认，等待安装完成
6. 安装进度会实时推送到前端显示

### 支持的 ModLoader

| ModLoader | 当前维护状态 | 支持的游戏版本 | 特点 |
|-----------|-------------|--------------|------|
| Fabric | 活跃维护 | 1.14+ | 轻量级，加载速度快，Mod 兼容性广，社区活跃 |
| Forge | 活跃维护 | 1.7.10+ | 老牌加载器，生态成熟，大型 Mod 支持完善 |
| NeoForge | 活跃维护 | 1.20.1+ | Forge 的新继任者，面向新版本重构，性能更优 |
| Quilt | 活跃维护 | 1.14+ | Fabric 的分叉，更模块化设计，提供额外 API 扩展 |

::: tip 选择建议
- **新手玩家**：推荐 Fabric，安装简单，Mod 兼容性好。
- **大型整合包玩家**：推荐 Forge 或 NeoForge，大型 Mod 支持更完善。
- **追求最新版本**：推荐 NeoForge，对新版本支持最快。
- **高级用户**：可以尝试 Quilt，享受更模块化的 API 设计。
:::

::: warning 注意
- 不同版本的 ModLoader 仅支持对应版本范围的游戏，安装前请确认兼容性。
- 同一个游戏版本只能安装一种 ModLoader。如需切换，请先卸载当前 ModLoader 再重新安装。
- 安装 ModLoader 后，启动器会自动创建对应的配置文件和目录结构。
:::

## 删除版本

如果你不再需要某个游戏版本，可以将其删除以释放磁盘空间。

### 删除步骤

1. 进入 **版本** 页面
2. 找到要删除的已安装版本
3. 点击该版本旁的 **「删除」** 按钮
4. 在确认对话框中点击「确认」

### 删除注意事项

| 项目 | 是否删除 | 说明 |
|------|---------|------|
| 游戏核心文件（JAR/JSON） | 是 | 版本目录下的核心文件会被移除 |
| 该版本的 ModLoader | 是 | 关联的 ModLoader 文件也会被清理 |
| 实例数据 | 否 | 实例系统中的数据不受影响 |
| 存档文件 | 视情况而定 | 如果启用了版本隔离，该版本的存档也会被删除 |
| 共享资源文件 | 否 | assets 和 libraries 目录中的共享文件不会被删除 |

::: warning 重要提醒
删除版本前，请确认该版本下没有重要的存档数据。建议先备份存档，再执行删除操作。
:::
