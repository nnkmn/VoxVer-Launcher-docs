# 实例系统

实例系统是 VoxVer Launcher 的核心功能之一，它允许你为不同的玩法创建完全独立的 Minecraft 运行环境。每个实例拥有独立的游戏版本、Mod、配置和存档，互不干扰。本章将详细介绍实例的创建、导入、导出以及隔离机制。

## 创建实例

通过创建实例，你可以为不同的游戏场景建立独立的环境，例如一个用于原版生存、一个用于 Mod 冒险、一个用于创造模式测试。

### 创建步骤

1. 进入 **实例** 页面
2. 点击 **「新建实例」** 按钮
3. 在创建向导中填写以下信息：
   - **实例名称** -- 为实例起一个便于识别的名称（如「生存整合包」、「测试环境」）
   - **游戏版本** -- 从下拉列表中选择基础游戏版本
   - **ModLoader** -- 选择要安装的 ModLoader（可选，也可以创建后再安装）
   - **ModLoader 版本** -- 选择 ModLoader 的具体版本号
4. 点击 **「创建」** 按钮
5. VoxVer Launcher 会自动创建实例目录结构并安装所选版本和 ModLoader

::: tip 提示
创建完成后，你可以在「实例详情」页面中进一步管理该实例的 Mod、配置和其他设置。
:::

## 导入实例

如果你已经在其他启动器中使用过 Minecraft，或者有一个现成的 `.minecraft` 目录，可以将其导入为 VoxVer Launcher 实例。

### 导入步骤

1. 进入 **实例** 页面
2. 点击 **「导入」** 按钮
3. 在文件选择器中浏览并选择已有的 `.minecraft` 目录（或其子目录）
4. VoxVer Launcher 会自动扫描目录中的游戏版本、Mod 和配置文件
5. 确认导入信息后，点击「确认」
6. 等待导入完成

::: info 说明
导入过程中，VoxVer Launcher 会识别目录中已有的游戏版本和 ModLoader，并自动创建对应的实例配置。原有的游戏文件不会被移动或修改。
:::

### 外部启动器数据导入

自 v0.6.3 起，VoxVer Launcher 支持自动检测并导入来自 **HMCL (Hello Minecraft! Launcher)** 和 **PCL2 (Plain Craft Launcher 2)** 的游戏实例数据。

1. 进入 **实例** 页面
2. 点击 **「导入」** 按钮，选择「从外部启动器导入」
3. VoxVer Launcher 会自动扫描系统中已安装的 HMCL 和 PCL2 数据目录
4. 识别到的实例列表将显示实例名称、游戏版本、ModLoader 类型和 Mod 数量
5. 勾选要导入的实例，点击「确认导入」
6. 导入完成后，实例将出现在 VoxVer Launcher 的实例列表中

::: tip 优势
- **无缝迁移** — 一键将 HMCL 或 PCL2 的实例数据迁移到 VoxVer Launcher
- **保留原有数据** — 原有启动器的数据不会被修改或删除
- **智能识别** — 自动解析实例的游戏版本、ModLoader 和 Mod 信息
:::

## 导出实例

你可以将实例打包导出，方便分享给其他玩家或备份存档。

### 导出步骤

1. 进入 **实例详情** 页面
2. 点击 **「导出」** 按钮
3. 选择导出路径和文件名
4. 选择导出内容（可选）：
   - 是否包含存档文件
   - 是否包含 Mod 文件
   - 是否包含配置文件
5. 点击「确认」，等待打包完成

::: tip 提示
导出的实例压缩包可以分享给其他 VoxVer Launcher 用户，对方通过「导入」功能即可使用。也可以作为备份存档的手段。
:::

## 实例隔离说明

实例系统的核心优势在于 **完全隔离**。每个实例都是一个独立的 Minecraft 运行环境，拥有自己的文件目录。

### 隔离内容

| 隔离项 | 目录 | 说明 |
|--------|------|------|
| 游戏版本 | `versions/` | 每个实例使用独立的游戏版本 JAR 和 JSON 文件 |
| Mod 文件 | `mods/` | 每个实例的 Mod 互不影响，不同实例可以安装不同版本的同一 Mod |
| 配置文件 | `config/` | 每个实例的 Mod 配置独立存储，修改一个实例的配置不会影响其他实例 |
| 存档数据 | `saves/` | 每个实例的存档完全独立，不会出现存档冲突 |
| 资源包 | `resourcepacks/` | 每个实例可以使用不同的资源包组合 |
| 日志文件 | `logs/` | 每个实例的运行日志独立记录，便于排查问题 |

### 实例目录结构示例

```
instances/
  └── 我的世界生存/
      ├── versions/          # 游戏版本文件
      ├── mods/              # Mod 文件
      ├── config/            # Mod 配置
      ├── saves/             # 游戏存档
      ├── resourcepacks/     # 资源包
      └── logs/              # 运行日志
```

::: tip 使用建议
- **多版本玩家**：为每个常用游戏版本创建独立实例，避免版本切换时反复安装 Mod。
- **Mod 玩家**：为不同类型的整合包创建独立实例，如「优化整合」、「冒险整合」、「科技整合」。
- **测试玩家**：创建专门的测试实例，用于测试新 Mod 或新配置，确保不影响主力存档。
:::

## P2P 实例分享

VoxVer Launcher 内置 **P2P 实例分享** 功能，基于 **WebRTC 直连技术**，无需上传到任何服务器即可与好友分享你的游戏实例。

### 工作原理

```
发送方                          VoxVer 信令服务器                    接收方
   |                                  |                                |
   |-- 生成分享码（6 位） ---------->|                                |
   |<-- 分享码 ------------------------|                                |
   |                                  |<-- 输入分享码 ----------------|
   |                                  |-- 转发匹配请求 -------------->|
   |<-- 建立 WebRTC 直连通道 ---------|                                |
   |-- 加密传输实例数据 ---------------------------------------->     |
   |<-- 接收完成 ----------------------------------------------------|
```

### 分享实例

1. 进入 **实例详情** 页面
2. 点击 **「分享」** 按钮
3. 启动器会自动生成一个 **6 位分享码**（例如 `A1B2C3`）
4. 将分享码告诉你的好友
5. 等待好友接受连接，即可开始传输

### 接收实例

1. 进入 **实例** 页面
2. 点击 **「接收分享」** 按钮
3. 输入好友提供的 6 位分享码
4. 等待连接建立
5. 确认导入信息，完成接收

::: tip 快捷接收 - voxver:// 协议
好友可以直接通过链接分享给你：`voxver://share:分享码`（例如 `voxver://share:A1B2C3`）。点击链接后 VoxVer Launcher 将自动打开并进入接收页面，无需手动输入分享码。此功能基于 VoxVer Launcher 注册的自定义 URL 协议。
:::

### 分享选项

| 选项 | 说明 |
|------|------|
| 包含存档 | 是否包含 `saves/` 目录中的游戏存档 |
| 包含 Mod | 是否包含 `mods/` 目录中的 Mod 文件 |
| 包含配置 | 是否包含 `config/` 目录中的 Mod 配置 |
| 包含资源包 | 是否包含 `resourcepacks/` 目录 |

### 特点与安全

::: tip 核心优势
- **直连传输** -- 基于 WebRTC P2P 直连，传输速度快，不限速
- **无需服务器中转** -- 启动器仅做信令协商，实例数据不经过任何第三方服务器
- **隐私安全** -- 你的实例数据只会在你和好友之间直接传输
- **操作简便** -- 6 位分享码，输入即可建立连接
:::

::: warning 注意
- 分享码默认 **5 分钟内有效**，超时后需要重新生成。
- 双方需要同时在线才能完成分享。
- 接收方需要安装与分享方相同（或兼容）的游戏版本。
:::
