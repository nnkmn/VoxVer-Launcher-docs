# 常见问题

本章节汇总了用户在使用 VoxVer Launcher 过程中最常遇到的问题及其解决方案。如果你遇到的问题未在此列出，请参阅 [崩溃监控](/guide/crash) 或前往 [GitHub Issues](https://github.com/nnkmn/VoxVer-Launcher/issues) 提交反馈。

## 启动相关

### 启动时报「无法找到 Java」？

**原因**：VoxVer Launcher 无法在系统中检测到有效的 Java 运行时环境。

**解决方案**：

1. 进入 **设置** 页面，查看「Java 设置」区域
2. 点击「自动检测」按钮重新扫描系统中的 Java
3. 如果仍未检测到，请手动指定 Java 路径：
   - 点击「浏览」，导航到 Java 安装目录的 `bin` 文件夹
   - 选择 `java.exe` 文件
   - 保存设置
4. 如果系统中尚未安装 Java，请前往 [Eclipse Temurin](https://adoptium.net/) 下载安装

详见 [Java 设置](/guide/java)。

### 游戏启动后闪退？

**原因**：闪退可能有多种原因，包括内存不足、Mod 冲突、Java 版本不匹配等。

**排查步骤**：

1. 查看 **崩溃监控** 中的崩溃报告，了解具体错误类型
2. 根据错误类型尝试以下解决方案：

| 错误类型 | 解决方案 |
|---------|---------|
| `OutOfMemoryError` | 增大最大内存设置至 4096MB 或更高 |
| Mod 相关错误 | 禁用所有 Mod 后尝试启动，再逐个启用排查冲突项 |
| `UnsupportedClassVersionError` | 安装与游戏版本匹配的 Java 版本 |
| 原生库错误 | 重新安装游戏版本 |

详见 [崩溃监控](/guide/crash)。

### 启动按钮灰色无法点击？

**可能原因**：

- 未选择游戏账户 -- 请先在「账户」页面添加账户
- 未安装游戏版本 -- 请先在「版本」页面安装至少一个版本
- 正在下载或安装中 -- 请等待当前操作完成

## 账户相关

### 微软登录卡住不动？

**可能原因及解决方案**：

| 原因 | 解决方案 |
|------|---------|
| 设备码输入错误 | 仔细核对启动器显示的设备码，确保输入正确 |
| 浏览器登录了错误的微软账号 | 在浏览器中退出当前微软账号，重新登录正确的账号 |
| Token 获取延迟 | 耐心等待 1-2 分钟，网络延迟可能导致 token 获取变慢 |
| 浏览器被拦截 | 检查防火墙或安全软件是否拦截了浏览器访问微软服务器 |
| 微软服务暂时不可用 | 稍后重试，或查看 [Xbox Live 状态页](https://status.xboxlive.com/) |

### 离线模式能否玩正版服务器？

**不能。** 离线模式仅适用于以下场景：

- 单机游戏（Survival / Creative / Adventure）
- 局域网联机（打开对局域网开放）
- 支持离线验证的服务器（非常少见）

要连接正版服务器（如 Hypixel、2b2t 等），必须使用微软正版登录。

### 微软登录后提示「无效的令牌」？

**原因**：保存的访问令牌（token）已过期或被撤销。

**解决方案**：

1. 进入 **账户** 页面
2. 移除当前失效的微软账户
3. 重新进行微软登录（Device Code Flow）
4. 如果问题持续，请检查你的微软账户密码是否已更改

## Mod 相关

### 安装的 Mod 不生效？

请按以下清单逐项检查：

1. **确认 ModLoader 已安装** -- 进入「版本」页面，检查目标版本是否已安装对应的 ModLoader（Fabric / Forge / NeoForge / Quilt）
2. **确认 Mod 类型匹配** -- Fabric Mod 只能在 Fabric 环境中加载，Forge Mod 只能在 Forge 环境中加载
3. **确认游戏版本兼容** -- 检查 Mod 描述中标注的支持版本是否与你安装的游戏版本一致
4. **确认 Mod 已启用** -- 在实例详情中检查 Mod 的启用开关是否处于开启状态
5. **检查前置依赖** -- 部分 Mod 需要安装前置 Mod（如 Cloth Config、GeckoLib 等），请查看 Mod 的描述页面
6. **确认文件完整** -- 下载中断可能导致 Mod 文件不完整，尝试重新下载

### CurseForge / Modrinth 下载失败？

**可能原因及解决方案**：

| 原因 | 解决方案 |
|------|---------|
| 网络连接异常 | 检查网络连接是否正常 |
| 下载源不可用 | 进入「设置」切换下载源 |
| 被防火墙拦截 | 检查防火墙或安全软件设置 |
| 磁盘空间不足 | 清理磁盘空间后重试 |
| 仓库服务维护 | 稍后重试，CurseForge 和 Modrinth 偶尔会进行维护 |

## 其他

### 游戏存档保存在哪里？

存档位置取决于你的配置：

| 场景 | 存档路径 |
|------|---------|
| 默认配置 | `.minecraft/saves/` |
| 启用版本隔离 | `.minecraft/versions/<版本名>/saves/` |
| 使用实例系统 | 实例目录下的 `saves/` 文件夹 |
| 自定义游戏目录 | `<自定义路径>/saves/` |

::: tip 提示
建议定期备份存档文件夹，防止意外数据丢失。
:::

### 如何完全卸载 VoxVer Launcher？

如果你决定卸载 VoxVer Launcher，请按以下步骤操作：

1. **卸载启动器程序**
   - 通过 Windows 控制面板的「程序和功能」卸载 VoxVer Launcher
   - 或运行安装程序选择「卸载」选项

2. **删除游戏数据**（可选）
   - 删除游戏目录（默认为 `.minecraft` 文件夹）
   - 注意：这会删除所有游戏版本、Mod、存档和配置

3. **删除启动器配置**（可选）
   - 删除 VoxVer Launcher 的应用数据目录（通常位于 `AppData/Roaming/VoxVer Launcher/` 或 `AppData/Local/VoxVer Launcher/`）

::: warning 重要提醒
卸载前建议先 **导出重要实例** 作为备份，详见 [实例系统](/guide/instances) 的导出功能。
:::

### VoxVer Launcher 支持 macOS 或 Linux 吗？

当前版本仅支持 **Windows** 平台。macOS 和 Linux 版本正在开发计划中（计划在 v0.4.0+ 版本中提供），敬请期待。

### 如何更新 VoxVer Launcher？

- **安装版**：VoxVer Launcher 会自动检测新版本并提示更新，你也可以手动前往 [GitHub Releases](https://github.com/nnkmn/VoxVer-Launcher/releases) 下载最新版本覆盖安装。
- **便携版**：需要手动下载新版本并替换文件。建议保留配置文件，仅替换程序文件。
